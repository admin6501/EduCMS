# 🚀 iptables Port-Forward Manager

اسکریپت تعاملی Bash برای راه‌اندازی **پورت فورواردینگ** روی سرور لینوکسی (مثلاً سرور ایران تمیز) به یک سرور مقصد (مثلاً سرور خارجی که پنل **3x-ui** روی آن نصب است).

---

## ✨ امکانات

- ✅ منوی تعاملی و کاملاً فارسی
- ✅ فعال‌سازی خودکار IP Forwarding (به‌صورت دائمی در `sysctl.conf`)
- ✅ پشتیبانی از **تک پورت / چند پورت با کاما / رنج پورت**
- ✅ پشتیبانی از **TCP / UDP / هردو**
- ✅ امکان تعریف پورت مقصد متفاوت با پورت ورودی (Port Mapping)
- ✅ تشخیص خودکار اینترفیس شبکه
- ✅ ذخیره دائمی قوانین (`iptables-persistent` در Debian/Ubuntu  و  `iptables-services` در CentOS/RHEL)
- ✅ نمایش، حذف و پاک‌سازی قوانین
- ✅ تست اتصال به سرور مقصد

---

## 📥 نصب و اجرا (روی سرور ایران)

```bash
# دانلود اسکریپت (یا با scp/nano روی سرور بگذارید)
wget -O port-forward.sh https://YOUR_HOST/port-forward.sh
# یا فایل را با nano بسازید و محتوای port-forward.sh را پیست کنید
nano port-forward.sh

# دسترسی اجرا
chmod +x port-forward.sh

# اجرا با دسترسی root
sudo bash port-forward.sh
```

## 🧭 نحوه‌ی استفاده

پس از اجرا، منوی زیر نمایش داده می‌شود:

```
 1) افزودن قانون فوروارد جدید
 2) نمایش قوانین فعلی
 3) حذف یک قانون
 4) پاک‌کردن همه‌ی قوانین NAT
 5) تست اتصال به سرور مقصد
 6) ذخیره‌ی دستی قوانین
 0) خروج
```

### مثال: فوروارد پورت‌های 3x-ui

1. گزینه `1` را انتخاب کنید.
2. آی‌پی سرور خارجی (مقصد) را وارد کنید:  مثلاً `185.10.20.30`
3. حالت پورت `2` را انتخاب کنید (چند پورت با کاما)
4. وارد کنید: `443,2053,8443,54321`
5. پروتکل `3` (هردو TCP+UDP)
6. ذخیره می‌شود ✅

از این به بعد، کاربر با اتصال به:

```
IP_سرور_ایران:443
IP_سرور_ایران:2053
IP_سرور_ایران:8443
```

به‌صورت شفاف به سرور خارجی هدایت می‌شود.

## ⚠️ نکات مهم

### 1) آی‌پی واقعی کاربر دیده نمی‌شود
چون از `MASQUERADE` استفاده می‌کنیم، سرور 3x-ui آی‌پی **سرور ایران** را به‌عنوان مبدا می‌بیند، نه آی‌پی واقعی کاربر.

اگر می‌خواهید آی‌پی واقعی کاربر منتقل شود، باید از **TPROXY**, **GRE Tunnel**, **WireGuard**, یا روش‌های Layer-7 مثل **HAProxy با PROXY protocol** استفاده کنید — که در این اسکریپت پیاده‌سازی نشده‌اند.

### 2) فایروال سرور مقصد
روی سرور خارجی (3x-ui) مطمئن شوید آی‌پی سرور ایران در فایروال **اجازه‌ی اتصال** دارد. اگر در 3x-ui فایروال داخلی فعال است، آی‌پی سرور ایران را whitelist کنید.

### 3) فایروال سرور ایران (UFW / Firewalld)
اگر روی سرور ایران از UFW یا Firewalld استفاده می‌کنید، باید پورت‌های مورد نظر را روی این فایروال‌ها هم باز کنید:

```bash
# UFW
sudo ufw allow 443/tcp
sudo ufw allow 443/udp

# Firewalld
sudo firewall-cmd --permanent --add-port=443/tcp
sudo firewall-cmd --permanent --add-port=443/udp
sudo firewall-cmd --reload
```

### 4) چند سرور ایران (Load Balancing)
چون دو سرور ایران دارید، می‌توانید:
- روی هردو این اسکریپت را اجرا کنید (هر دو به همان سرور خارجی فوروارد کنند).
- در DNS یا کلاینت‌های V2Ray از هر دو آی‌پی به‌عنوان آدرس سرور استفاده کنید (Round-Robin).

### 5) پایداری بعد از ریبوت
اسکریپت به‌صورت خودکار:
- `net.ipv4.ip_forward=1` را در `/etc/sysctl.conf` می‌گذارد.
- قوانین را در `/etc/iptables/rules.v4` (Debian/Ubuntu) یا `/etc/sysconfig/iptables` (RHEL) ذخیره می‌کند.
- بسته‌ی `iptables-persistent` را نصب می‌کند.

## 🧪 تست سریع

پس از اعمال قوانین، از روی کلاینت (موبایل/کامپیوتر) با کلاینت v2rayNG / Hiddify اتصال V2Ray را با **آی‌پی سرور ایران** (به‌جای سرور خارجی) تنظیم کنید. اگر اتصال برقرار شد ✅، تمام شده است.

برای تست از خط فرمان:

```bash
# از یک سرور سوم
nc -zv IP_سرور_ایران 443
curl -v https://IP_سرور_ایران:2053
```

## 🛠️ عیب‌یابی

| مشکل | راه‌حل |
|------|--------|
| `iptables: command not found` | اسکریپت خودش نصب می‌کند، اگر نه: `apt install iptables` |
| اتصال timeout می‌شود | چک کنید فایروال سرور مقصد آی‌پی این سرور را block نکرده باشد |
| قوانین بعد از ریبوت پاک می‌شوند | بسته‌ی `iptables-persistent` نصب نشده — مجدداً اسکریپت را اجرا کنید |
| `Operation not permitted` | با `sudo` یا کاربر `root` اجرا کنید |

## 📝 پاک‌سازی دستی

اگر می‌خواهید همه چیز را reset کنید:

```bash
sudo iptables -t nat -F
sudo iptables -t nat -X
sudo iptables-save > /etc/iptables/rules.v4
```

یا از خود اسکریپت گزینه‌ی **4** را انتخاب کنید.
